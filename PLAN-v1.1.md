비전 재확인: **"백업이 아니라 관리"** — 빠르게 훑고, 정리하고(앨범/사람/중복), 필요한 것만 꺼내는 개인용 macOS 네이티브 앱.

## 3. 기능 계획 (주제별)

### A. 안정성 — 매일 쓰는 앱의 기본기
| 항목 | 내용 | 난이도 |
|---|---|---|
| A1 | 세션 자동 재로그인 (B1) | 중 |
| A2 | 오류 표시 체계 (B2+B10) — 배너 하나로 통일 | 하 |
| A3 | 동영상 업로드 허용 (B3) | 최하 |
| A4 | 삭제 시 스크롤 유지 (B4) | 하 |
| A5 | 썸네일 실패 placeholder (B7) | 하 |

### B. 관리 핵심 — 이 앱의 존재 이유 (차별화 기능)
| 항목 | 내용 | 비고 |
|---|---|---|
| B1' | **중복/유사 사진 정리** — API 이름이 `SYNO.Foto.Browse.SimilarItem`인 것 자체가 단서. 유사 그룹 나열 메서드 probe → 그룹별로 "최고 1장 남기고 정리" UI | **킬러 기능 후보**. probe 필요 |
| B2' | **스마트 정리 뷰** — 스크린샷(파일명 패턴), 대용량 동영상 Top N, 오래된 캡처 등 "용량 회수" 후보를 모아 보여주기 | 기존 필터/검색 조합으로 구현 가능 |
| B3' | **안전 삭제 옵션** — 휴지통이 없는 NAS 특성 보완: "다운로드(백업) 후 삭제" 원클릭 | 기존 download+delete 조합 |
| B4' | **지도 보기** — GPS 있는 사진을 MapKit 지도에 클러스터로 표시, 핀 클릭 → 그 장소 사진 | 인스펙터 지도의 확장. geocoding 필터 재사용 |
| B5' | **촬영일 수정** — 날짜 잘못 박힌 사진 교정 (`Browse.Item set`? probe 필요) | 관리 앱 단골 기능 |
| B6' | **즐겨찾기** — 별표 토글 + 필터 연동 (설정 API probe 필요; 현재 NAS 데이터 0) | 낮은 우선 |

### C. 완성도 — 반쪽 화면 채우기
| 항목 | 내용 |
|---|---|
| C1 | 앨범 상세 개선 (B5) + 앨범 이름 변경(API probe) |
| C2 | 인물 상세 개선 (B6) |
| C3 | 공유 토글 가드 (B8) + 중복 앨범 경고 (B9) |
| C4 | **썸네일 디스크 캐시** — 현재 메모리 800장뿐 → 재실행마다 전부 재다운로드. cache_key 기반 디스크 캐시로 재실행 즉시 표시 |
| C5 | Quick Look에서 **원본 지연 로드** (지금은 xl 썸네일) + 스크롤 줌 |

### D. Mac다움 — 네이티브 앱답게
| 항목 | 내용 |
|---|---|
| D1 | **메뉴 바 구성** — File(업로드/다운로드), Edit(⌘A 전체 선택), View(연도/월/일 전환 ⌘1/2/3, 필터 토글) 등 표준 메뉴+단축키 |
| D2 | **Finder로 드래그 내보내기** — 그리드에서 사진을 Finder로 끌면 원본 다운로드(파일 프로미스). "꺼내기"가 드래그 한 번 |
| D3 | 키보드 강화 — 그리드 2D 화살표 이동(현재 좌우 1D), Space 미리보기(있음), ⌘A |
| D4 | 설정 창(⌘,) — 연결 관리(다중 NAS 전환·삭제), 캐시 비우기, 기본 보기 단위, 로그아웃 |
| D5 | 앱 아이콘 제작 |
| D6 | 창 상태 복원(사이드바 선택·창 크기) |

### E. 성능
| 항목 | 내용 |
|---|---|
| E1 | 디스크 썸네일 캐시 (C4와 동일 — 성능 효과가 가장 큼) |
| E2 | 연도 점프 — 필터의 기간 프리셋("2024년"…)으로 옛날 사진 즉시 접근 (스크롤 전체 로드 불필요) |
| E3 | 삭제 로컬 반영 (A4와 동일) |

## 4. 진행 순서 (마일스톤)

```
M1  안정성        A3(1줄) → A2 오류 표시 → A1 세션 재로그인 → A4 → A5
M2  반쪽 채우기    C1 앨범 상세 → C2 인물 상세 → C3 가드들
M3  성능·캐시     C4/E1 디스크 캐시 → E2 연도 점프
M4  관리 킬러     B1' 중복 정리(probe부터) → B2' 스마트 정리 → B3' 안전 삭제
M5  Mac다움      D1 메뉴/단축키 → D2 드래그 내보내기 → D4 설정 창 → D5 아이콘
--  틈틈이        B4' 지도 보기, B5' 촬영일 수정, C5 원본 줌, D3/D6
```

선정 논리:
- **M1이 먼저**: 세션 만료·무음 실패는 "매일 쓰다 보면 반드시 걸리는" 문제. 신뢰가 우선.
- **M4(중복 정리)가 진짜 목표**: "관리 앱"의 존재 이유. 단, SimilarItem probe가 선행돼야 해서 M2~M3 뒤에 배치 (probe만 M2 중에 미리 해봐도 됨).
- 각 항목은 독립적이라 순서 조정 자유.

## 5. 하지 않기로 한 것 (기록)

- 배포/공증 패키징 (사용자 결정)
- 공유 링크 UI (제거됨 — `ShareSheet`/`ShareViewModel`/FotoService 메서드 보존, 버튼+`.sheet` 한 줄로 복원 가능)
- 태그 (NAS 태그 0개), 평점 필터 (데이터 0)
- 클라이언트 측 얼굴 인식 (서버가 이미 수행)
- 키워드 텍스트 검색 (검색 탭 제거 시 함께 제거 — 요청 시 필터 패널에 복원 가능)

## 6. 기술 부채 (조용히 갚을 것)

- Swift 6 strict-concurrency 마이그레이션 (현재 SWIFT_VERSION 5.0, project.yml에 명시된 추적 항목)
- FotoKitChecks 확장 — 새로 확정된 API(병합·set_cover·필터·공유)의 fixture 디코딩 체크 추가
- WriteSpike 정리 — probe 모드가 20개+ 누적, 확정된 것 제거
- `FotoError.notConfigured` 미사용 케이스 정리
